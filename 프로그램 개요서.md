# 프로젝트 개요서: 학습 밀착형 AI 공학계산기 (Edu-Math AI Calculator)

## 1. 프로젝트 개요 (Introduction)
**개발 배경:**
기존의 공학용 계산기는 단순히 정답만을 제공하여 학습자의 문제 해결력 향상과 원리 이해에 한계가 존재합니다. 이를 해결하기 위해 단순 결과 도출을 넘어, 문제 풀이 과정과 필수 수학적 개념을 학습자의 눈높이에 맞추어 설명해주는 '학습 밀착형 AI 공학계산기'를 기획하였습니다.

**타겟 사용자:**
- 초등/중학생: 수식의 기본 개념과 원리 이해가 필요한 기초 학습자
- 고등학생: 내신 및 수능 대비를 위해 다양한 유형의 유사 문제 풀이가 요구되는 심화 학습자
- 대학생/일반인: 전공 수학 등 복잡한 수식의 단계별 도출 과정이 필요한 고급 학습자

---

## 2. 핵심 기능 상세 (Key Features)
- **수식 계산 및 결과 출력:** 사용자가 입력한 수식에 대해 정확한 수학적 결과값을 산출합니다.
- **단계별 풀이 과정 및 핵심 개념 설명:** 결과 도출까지의 모든 과정을 단계별로 분할하여 제시하며, 각 단계에 쓰인 핵심 수학 개념을 설명합니다.
- **학습자 수준별 맞춤 설명:**
  - *초등/중등:* 직관적이고 쉬운 비유를 활용한 원리 설명
  - *고등:* 교육과정과 연계된 공식 증명 및 개념 설명
  - *대학생:* 엄밀한 수학적 정의와 이론적 배경을 포함한 심층 해설
- **학습 강화용 유사 문제 생성:** 입력한 수식 및 사용된 개념과 유사한 난이도의 문제를 자동 생성하여 완전 학습을 돕습니다.

---

## 3. 기술 스택 (Tech Stack)
| 분류 | 기술 (Technologies) | 도입 이유 |
|---|---|---|
| **Frontend** | HTML5, CSS3, Vanilla JS | 경량화된 웹 구현 및 사용자 입력 최적화, 불필요한 의존성 최소화 |
| **Backend** | Python (Flask) | Vercel Serverless Function 배포가 용이하며, Groq API 및 수학 라이브러리와의 연동이 직관적임 |
| **AI Engine** | Groq AI API | 초고속 추론(Inference) 라우팅을 통해 실시간 수식 해석 및 해설 생성 |
| **Infra** | Git, Vercel | 버전 관리 및 지속적 통합/배포(CI/CD) 자동화, 유지보수 비용 최소화 |

---

## 4. 시스템 아키텍처 (Architecture)
다음은 클라이언트(Frontend), 서버(Backend), 그리고 인공지능(Groq API) 간의 데이터 흐름 시퀀스입니다.

1. **입력(Client):** 사용자가 웹 브라우저(HTML/JS)에서 계산할 수식과 본인의 학습 수준(초등/중등/고등/대학생)을 선택하고 요청(Request)을 전송합니다.
2. **처리(Backend):** Vercel에 배포된 Flask 서버가 요청을 수신합니다. Flask 서버는 입력된 데이터를 기반으로 프롬프트를 구성하여 Groq AI API를 호출합니다.
3. **AI 추론(AI Engine):** Groq API는 수식의 정답, 단계별 풀이, 맞춤형 설명, 유사 문제를 생성하여 JSON 형태로 Flask 서버에 반환(Response)합니다.
4. **출력(Client):** Flask는 받은 데이터를 정제하여 프론트엔드로 전달하고, Vanilla JS가 이를 화면에 렌더링(Rendering)합니다. 수식은 LaTeX 포맷으로 포매팅되어 렌더링됩니다.

## 5. 상세 파일 구조 (Project Structure)
Vercel 배포 및 Serverless 환경을 고려한 Flask 기반의 프로젝트 디렉토리 구조입니다.

```text
edu-math-ai-calculator/
├── api/
│   └── index.py            # Vercel Serverless Function 진입점 (Flask 앱 초기화 및 라우팅)
├── static/
│   ├── css/
│   │   └── style.css       # 반응형 및 UI 스타일링
│   └── js/
│   │   └── app.js          # 프론트엔드 비동기 통신(Fetch) 및 데이터 렌더링
├── templates/
│   └── index.html          # 메인 UI 구조 
├── requirements.txt        # Python 패키지 의존성 목록 (Flask, groq 등)
├── vercel.json             # Vercel 배포 설정 파일
└── README.md               # 프로젝트 개요 및 실행 가이드
```

## 6. UI/UX 가이드라인
- **직관적인 입력 영역:** 수식 입력창과 학습자 수준(난이도) 선택 드롭다운을 상단에 고정하여 접근성 향상.
- **수식 가독성 극대화:** 서버로부터 전달받은 모든 수학 기호와 수식은 MathJax 또는 KaTeX와 같은 라이브러리를 활용해 LaTeX 문법(`$...$` 또는 `$$...$$`)으로 변환 후 렌더링.
- **단계별 아코디언(Accordion) UI:** 긴 풀이 과정이나 추가 유사 문제는 접고 펼칠 수 있는 아코디언 UI를 채택하여 인지 부하 감소.
- **반응형 웹 디자인(RWD):** 모바일, 태블릿, 데스크톱 등 다양한 디바이스 해상도에 최적화된 화면 제공.

---

## 7. 향후 확장성
- **그래프 시각화:** 함수식 입력 시 2D/3D 그래프를 렌더링하는 시각화 도구(Desmos API 또는 Chart.js 등) 연동 기능.
- **학습 이력 저장 및 오답 노트:** 사용자 계정 연동을 통한 과거 계산 이력 저장, 틀린 유사 문제 등을 스크랩할 수 있는 오답 노트 기능 제공.
- **OCR(광학 문자 인식) 기능:** 수식을 직접 타이핑하지 않고 사진 촬영만으로 입력할 수 있는 이미지 텍스트 추출 기능 추가.
